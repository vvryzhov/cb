# Генерация тестовых данных

## Описание

Скрипт генерирует тестовые данные для проекта Payroll BI:
- 8 департаментов
- Отделы для каждого департамента
- Группы для отделов
- 100 сотрудников (по умолчанию, можно изменить)
- История изменений зарплаты для каждого сотрудника
- Назначенные руководители

## Использование

### Вариант 1: Без очистки существующих данных

```bash
python manage.py generate_test_data
```

### Вариант 2: С очисткой существующих данных

```bash
python manage.py generate_test_data --clear
```

### Вариант 3: Указать количество сотрудников

```bash
python manage.py generate_test_data --employees 200
```

### Вариант 4: Очистить и создать 100 сотрудников

```bash
python manage.py generate_test_data --clear --employees 100
```

## Параметры

- `--employees N` - количество сотрудников для генерации (по умолчанию 100)
- `--clear` - очистить существующие данные перед генерацией

## Что генерируется

### Департаменты:
- Разработка
- Продажи
- Маркетинг
- HR
- Финансы
- Операции
- Поддержка
- Аналитика

### Отделы:
Для каждого департамента создаются соответствующие отделы (например, для "Разработка": Backend, Frontend, Mobile, DevOps)

### Группы:
Для каждого отдела создается 1-2 группы

### Сотрудники:
- Случайные ФИО (русские имена)
- Логины вида `user_001`, `user_002`, и т.д.
- Случайные должности
- Случайная организационная структура
- Дата принятия от 1 до 5 лет назад
- Финансовые показатели:
  - Оклад: 50,000 - 500,000 ₽
  - Квартальная премия: 0 - 50% от оклада
  - Месячная премия: 0 - 30% от оклада
  - Годовая премия: 0 - 200% от оклада

### История зарплаты:
Для каждого сотрудника создается 1-3 записи истории изменений зарплаты с датами от даты приема до сегодня

## Примеры использования

### Создать 100 сотрудников (без очистки)
```bash
python manage.py generate_test_data
```

### Очистить все и создать 150 сотрудников
```bash
python manage.py generate_test_data --clear --employees 150
```

### Создать только 50 сотрудников
```bash
python manage.py generate_test_data --employees 50
```

## После генерации

После выполнения команды вы сможете:
1. Просмотреть сотрудников на странице `/employees/`
2. Посмотреть аналитику на странице `/analytics/`
3. Создать произвольные отчеты на странице `/custom-report/`
4. Использовать данные в Saiku для BI-аналитики

## Примечания

- Данные генерируются случайным образом
- Руководители назначаются случайно из существующих сотрудников
- История зарплаты показывает постепенное увеличение от даты приема до текущих значений
- Все сотрудники создаются как активные (`is_active=True`)
